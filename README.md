# CityOpsDB - Σύστημα Διαχείρισης Αστικών Περιστατικών

Αυτό το αποθετήριο περιέχει τη λογική σχεδίαση και την πλήρη υλοποίηση σε SQL μιας βάσης δεδομένων για τη διαχείριση περιστατικών σε έναν Δήμο. Το project αναπτύχθηκε για το μάθημα **«Βάσεις Δεδομένων»** (9ο Εξάμηνο, 2025-2026) του Τμήματος ΗΜΜΥ ΑΠΘ.

## Περιγραφή Έργου
Το **CityOpsDB** επιτρέπει στους πολίτες να αναφέρουν προβλήματα στην πόλη (π.χ. λακκούβες, καμένες λάμπες) και στον Δήμο να οργανώνει την επίλυσή τους μέσω των αρμόδιων τμημάτων και υπαλλήλων. Το σύστημα υποστηρίζει την πλήρη καταγραφή της πορείας κάθε περιστατικού, το ιστορικό καταστάσεων, τη γεωγραφική κατανομή ανά περιοχή, καθώς και την ανάρτηση φωτογραφικού υλικού.

## Δομή Αποθετηρίου
* `create_schema.sql`: Δημιουργία των πινάκων, των ξένων κλειδιών και των περιορισμών ακεραιότητας (3NF).
* `insert_data_into_tables.sql`: Εισαγωγή ρεαλιστικών δεδομένων για δοκιμή του συστήματος.
* `creates_views.sql`: Δημιουργία Όψεων (Views) για εύκολη πρόσβαση σε σύνθετες πληροφορίες.
* `users.sql`: Ορισμός ρόλων χρηστών (Admin, Employee, Citizen) και δικαιωμάτων πρόσβασης (RBAC).
* `queries/`: Συλλογή SQL ερωτημάτων για εξαγωγή στατιστικών και αναζήτηση δεδομένων.
* `cityopsdb.sql`: Πλήρες SQL Dump της βάσης δεδομένων.

## Σχήμα Βάσης Δεδομένων
Η βάση αποτελείται από 10+ πίνακες, συμπεριλαμβανομένων των:
- **Citizen / Employee / Department**: Διαχείριση ανθρώπινου δυναμικού και χρηστών.
- **Incident / IncidentType / District**: Στοιχεία και κατηγοριοποίηση των περιστατικών.
- **StatusHistory**: Πλήρες ιστορικό μεταβολών κατάστασης (ΝΕΟ, ΣΕ ΕΞΕΛΙΞΗ, ΟΛΟΚΛΗΡΩΘΗΚΕ κ.λπ.).
- **Action / Assignment**: Καταγραφή των εργασιών που εκτελούνται για κάθε αίτημα.

## Οδηγίες Εγκατάστασης
Για να στήσετε τη βάση δεδομένων τοπικά:

1. Ανοίξτε το **MySQL Workbench** ή οποιοδήποτε τερματικό MySQL.
2. Εκτελέστε το αρχείο `create_schema.sql` για να δημιουργηθεί η δομή.
3. Εκτελέστε το αρχείο `creates_views.sql` για τις όψεις.
4. Εκτελέστε το αρχείο `insert_data_into_tables.sql` για τα δεδομένα.
5. (Προαιρετικά) Εκτελέστε το `users.sql` για να δημιουργηθούν οι ρόλοι χρηστών.

## Παραδείγματα Ερωτημάτων
Το σύστημα μπορεί να απαντήσει σε ερωτήματα όπως:
- Πόσα περιστατικά ολοκλήρωσε το "Τμήμα Καθαριότητας" μέσα στο 2025;
- Ποια είναι η τρέχουσα κατάσταση των αιτημάτων που υπέβαλε ένας συγκεκριμένος πολίτης;
- Ποιοι υπάλληλοι είναι υπεύθυνοι για ενέργειες με υψηλό κόστος;
